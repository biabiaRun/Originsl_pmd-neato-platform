1 Spectre API {#spectre_api}
2 ================
3 
4 \warning
5 
6 **The major architectural changes in Spectre 4 are not reflected in
7 the documentation. Updates to the documentation are planned for future
8 Spectre 4 releases.**
9 
10 
11 \picture{api_fig1, api_interface.png, "Spectre's Public C++ API"}
12 
13 The Spectre C++ public API is a intented as the single public entry for computing
14 depth data using Spectre. Users of the Spectre C++ public API do not need to
15 use any functions from libcalibration, or configure libprocessing manually,
16 because the API will take care of this. Additionally, it acts as an
17 abstraction layer which hides the differences between platform specific
18 libprocessing implementations. Furthermore, in the future some feature might
19 only be available through C++ API. Therefore, users are encouraged to use the
20 Spectre C++ public API instead of accessing libcalibration / libprocessing
21 directly.
22 
23 As outlined in the class diagram above, the Spectre public C++ API consists of
24 three main classes in the namespace `::spectre`:
25 1. The interface `spectre::ISpectre` which is the main entry point of the Spectre
26  public API. An instance of the interface can be obtained through the
27  factory function `spectre::getSpectre` which requires the calibration data block as
28  a mandotary argument. Each `spectre::ISpectre` instance is coupled with the
29  calibration data block used during its creation.
30 2. The `Input` class which is used to pass the sensor raw input data to
31  Spectre. This class contains only data which (possibly) changes for every
32  raw frame.
33 3. The `Configuration` class which holds the state of Spectre's
34  configuration. A configuration consist of `spectre::Parameter`s and (optional)
35  output redirections. `spectre::Parameter`s allow to configure Spectre's
36  behaviors. Depending on the calibration data, and libprocessing
37  implementation used different sets of parameters are available, which can
38  be queried through the `spectre::Configuration` instance. Redirection are used to
39  tell Spectre where to store the depth-computation results. If not used,
40  Spectre uses memory managed by Spectre for computation results Each
41  `spectre::Configuration` instance is coupled to a specific `spectre::ISpectre` instance. Use
42  the function `ISpectre::configuration()` to obtain a copy of the current
43  configuration for modification. You can apply the changed `spectre::Configuration`
44  instance through spectre::ISpectre::reconfigure.
45 
46 The remaining classes are utility classes which
47 - represent a parameter in a configuration (`spectre::Parameter`)
48 - allow to pass existing memory to Spectre (`spectre::ArrayReference`)
49 - allow to pass memory hold by the instance to Spectre (`spectre::ArrayHolder`)
50 
51 Their usage should become clear from the example below. For a detailed
52 documentation of the individual API functions, and data structures consult the
53 `::spectre` namespace documentation.

56 Spectre Usage
57 -------------
58 
59 The following example code can be used as starting point for computing
60 depth-data from sensor raw data:
61 
62 ~~~~~~~~~~~{.cpp}
63 #include <algorithm>
64 
65 #include <spectre/ISpectre.hpp>
66 
67 int main(int argc, char *argv[])
68 {
69  using namespace spectre;
70 
71  // Initialize this vector with your calibration data
72  std::vector<uint8_t> calibBlob;
73  ISpectre *spectre = nullptr;
74  auto status = getSpectre(ArrayReference<uint8_t>(calibBlob.data(), calibBlob.size()), &spectre);
75  if (status)
76  {
77  // Set desired options (e.g., switch to 5 - Phase mode)
78  auto config = spectre->configuration();
79  auto numFreqs = config.getParameterByKey(ParameterKey::NUM_FREQUENCIES);
80  auto freq1 = config.getParameterByKey(ParameterKey::FREQUENCY_1);
81  numFreqs.setValue(ParameterVariant(static_cast<uint32_t>(1u)));
82  // Set YOUR modulation frequency here
83  freq1.setValue(ParameterVariant(static_cast<uint32_t>(6024000u)));
84  if (!(config.setParameter(numFreqs) && config.setParameter(freq1) && spectre->reconfigure(config)))
85  {
86  throw "Could not reconfigure Spectre";
87  }
88 
89  // Initialize these values with the sensor raw data / exposure times / illumination temperature
90  std::vector<uint16_t> intensityImage, rawFrame1, rawFrame2, rawFrame3, rawFrame4;
91  unsigned intensityExp = 100u, modExp = 500u;
92  float illuTemp = 30.0f;
93 
94  Input<ArrayReference> input(2u);
95  input.setIntensityFrame(ArrayReference<uint16_t>(intensityImage.data(), intensityImage.size()), intensityExp);
96  input.setModulatedFrames<ParameterKey::FREQUENCY_1>(ArrayReference<uint16_t>(rawFrame1.data(), rawFrame1.size()),
97  ArrayReference<uint16_t>(rawFrame2.data(), rawFrame2.size()),
98  ArrayReference<uint16_t>(rawFrame3.data(), rawFrame3.size()),
99  ArrayReference<uint16_t>(rawFrame4.data(), rawFrame4.size()),
100  modExp);
101  input.setTemperature(illuTemp);
102 
103  if (spectre->processWhole(input))
104  {
105  // Reference to computed depth-point cloud (x,y,z,c)
106  auto pointCloud = spectre->results<ResultType::COORDINATES>();
107 
108  // Do something (e.g., copy to a std::vector)
109  std::vector<float> xyzc;
110  std::copy (pointCloud.cbegin(), pointCloud.cend(), std::back_inserter(xyzc));
111  }
112  else
113  {
114  throw "Could not process data";
115  }
116 
117  delete spectre;
118  }
119 
120  return 0;
121 }
122 ~~~~~~~~~~~
